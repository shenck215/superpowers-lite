# Superpowers Lite

Superpowers Lite 是一个轻量级 Codex skill，用于在不引入沉重全局流程的前提下，为工程任务提供更结构化、更可执行的处理方式。

它会根据任务复杂度自动选择最轻流程：小任务直接给结果，中等任务补充必要分析和方案，高风险或跨模块任务再进入更完整的复核流程。

## 适用场景

- 软件设计、接口开发、代码重构或复杂问题排查。
- 希望回答更工程化，但不想每次都触发完整流程。
- 希望按任务复杂度控制分析深度和 token 消耗。
- 希望在实现类请求中优先获得具体结果、代码或最小补丁。

## 启用方式

在请求中显式调用：

```text
$superpowers-lite 帮我重构这个函数，减少重复判断
```

或者：

```text
使用 $superpowers-lite 处理这个工程任务，并按复杂度选择最轻流程。
```

该 skill 默认不支持隐式调用，只有用户明确提到 `$superpowers-lite` 时才会启用。

## 工作模式

### Quick

用于局部、明确、低风险的任务。

典型例子：

- 修改一个函数。
- 修复一个范围很小的 bug。
- 给接口补一个简单参数。

输出风格：

- 直接给出结果。
- 只保留必要解释。
- 必要时简短说明验证方式。

### Standard

用于需要一定结构化处理，但还不需要完整 review 闭环的任务。

典型例子：

- 涉及多个相关文件。
- 需要接口契约、分页、校验或权限逻辑。
- 实现前需要少量取舍分析。

输出风格：

- 简短分析问题与约束。
- 给出最小可靠方案。
- 直接实现或给出具体实现步骤。

### Deep

用于高风险、跨模块或定义不充分的任务。

典型例子：

- 架构设计或子系统设计。
- 大规模重构。
- 线上问题根因分析。
- 影响可靠性、安全性、并发、迁移或性能敏感路径。

输出风格：

- 界定问题和失败模式。
- 说明约束与取舍。
- 给出推荐方案和实现路径。
- 复核风险、缺口和验证需求。

## 复杂度路由

当任务处在边界情况时，会参考 `references/routing-guide.md`，从以下维度评分：

- Scope：影响范围。
- Ambiguity：需求歧义。
- Risk：行为或系统风险。
- Diagnosis：是否需要排查根因。

评分越高，流程越完整。若任务涉及认证、授权、支付、数据删除、迁移、并发或外部兼容性，会自动提高处理严谨度。

## 目录结构

```text
.
├── SKILL.md
├── agents/
│   └── openai.yaml
└── references/
    └── routing-guide.md
```

说明：

- `SKILL.md`：skill 的主说明文件，定义触发条件、核心规则、复杂度模式和输出形态。
- `agents/openai.yaml`：Codex 展示名称、简介和默认提示词配置。
- `references/routing-guide.md`：复杂度评分规则和边界案例。

## 安装

将本目录放到 Codex skills 目录下即可，例如：

```text
~/.codex/skills/superpowers-lite
```

安装后，在 Codex 中显式使用 `$superpowers-lite` 即可触发该 skill。

## 设计原则

- 只在用户显式调用时启用。
- 先选择满足质量要求的最轻流程。
- 小任务不过度设计。
- 中等任务保持分析简洁，并尽快进入执行。
- 高风险任务补足复核、风险和验证说明。
- 当用户要求实现时，默认交付具体结果，而不是停留在建议。
